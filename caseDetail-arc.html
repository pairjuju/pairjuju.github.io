新思科技- ARC智慧物聯網開發平台

實際網站 website
  1. 專案背景 Project Overview

    新思科技欲製作一個產品開發網站，以公布產品使用指南及影音教學。並且希望學生、教職員可利用該平台討論，提出產品使用上的問題，讓網站管理人員、公司內部工程師可針對問題提出解答。

    另，每年度該公司會舉辦旗下產品相關的比賽活動，所以該網站也需要有報名活動、審核報名等功能。

  2. 我的角色 My role
    UI/UX Designer Project Manager
  3. 負責項目 Deliverables
    • 專案管理，負責與客戶說明討論介面流程和功能細項，並與工程師描述客戶需求詢問實作可行性(ppt+adobe XD)
    • 規劃前台頁面(RWD)
    • 後台CMS介面流程設計(11項主功能，24項次功能)
    • 使用bootstrap/html/css/jQuery，切版前後台頁面(RWD)
    • 撰寫前後台操作手冊，共85頁(.word)
    • 蒐集客戶回饋，整理待修改項目(.excel)
    • 維護保固，修改跑版頁面
    • 功能測試、查找重現bug並整理成待修改列表(.excel)
  4. 設計目標 Design Goal
    • 【前台網站】
      讓User能於各裝置使用前台網站順利註冊，報名活動，觀看教學影音與下載文件，討論區發表文章，留言回覆文章等等。
    • 【後台CMS】
      讓網站管理人員建立活動，活動報名表單除提供基本固定欄位外，也提供管理員可新增欄位，自定義報名表單；也提供報名審核、取消報名審核等等功能。 為避免討論區有垃圾訊息或是不當留言，後台提供管理員前台討論區管理功能，可上架/下架/刪除/編輯文章及留言。
  5. 過程與挑戰 Process
    • 雖然活動報名功能很常見，但由於每家公司的處理方式都會有少許不同，所以儘管客戶提出了該功能需求，仍需使用ppt+簡易flow chart確認細部流程(審核狀態的變化及通知信件的發送)，才能製作合適的管理後台及處理程序。
